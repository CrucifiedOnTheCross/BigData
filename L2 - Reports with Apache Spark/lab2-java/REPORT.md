# Отчет по лабораторной работе 2

## Тема

Формирование отчетов в Apache Spark. Анализ популярности языков программирования по данным Stack Overflow.

## Цель работы

Цель лабораторной работы - научиться формировать аналитические отчеты с помощью Apache Spark и сохранять результат в формате Apache Parquet.

## Постановка задачи

Необходимо сформировать отчет о 10 наиболее популярных языках программирования по итогам каждого года за период с 2010 по 2020 год.

В качестве входных данных используются:

- `posts_sample.xml` - выборка постов Stack Overflow;
- `programming-languages.csv` - список языков программирования.

Итоговый отчет должен быть сохранен в формате Apache Parquet.

## Выбранный подход

Решение выполнено на Java с использованием Apache Spark SQL.

Spark запускается в локальном режиме внутри JVM-процесса. Отдельный Spark-кластер не поднимается. Такой подход подходит для лабораторной работы и позволяет проверить всю цепочку обработки данных: чтение XML, преобразование, агрегацию, ранжирование и запись Parquet.

Spring Boot не использовался, потому что задача является batch-обработкой данных, а не серверным приложением.

## Структура проекта

Проект расположен в папке:

```text
L2 - Reports with Apache Spark/lab2-java
```

Основные файлы:

- `pom.xml` - Maven-конфигурация;
- `README.md` - инструкция по запуску;
- `src/main/java/ru/bigdata/lab2/Lab2Application.java` - точка входа;
- `src/main/java/ru/bigdata/lab2/PopularityReportService.java` - построение отчета;
- `src/main/java/ru/bigdata/lab2/StackOverflowXmlParser.java` - парсинг XML-строк;
- `src/main/java/ru/bigdata/lab2/StackOverflowXmlReader.java` - чтение XML в Spark;
- `src/main/java/ru/bigdata/lab2/LanguageDictionary.java` - загрузка справочника языков;
- `src/main/java/ru/bigdata/lab2/NoOpPermissionLocalFileSystem.java` - обход проблемы `winutils.exe` при записи Parquet на Windows;
- `src/main/java/ru/bigdata/lab2/model` - модели данных.

## Описание реализации

### Чтение Stack Overflow XML

Файл `posts_sample.xml` содержит XML-строки постов Stack Overflow. Для лабораторной работы нужны только вопросы, поэтому приложение отбирает записи, у которых:

```text
PostTypeId = 1
```

Для каждого вопроса извлекаются:

- идентификатор;
- дата создания;
- год;
- теги.

### Разбор тегов

Теги Stack Overflow представлены строкой вида:

```text
<java><spring><spark>
```

Парсер выделяет отдельные теги и сопоставляет их со справочником языков программирования.

### Справочник языков

Файл `programming-languages.csv` используется как источник допустимых языков программирования.

Класс `LanguageDictionary` загружает список языков и позволяет сопоставлять теги вопросов с языками. Это нужно, чтобы не считать произвольные теги Stack Overflow, которые не являются языками программирования.

### Расчет популярности

Популярность языка определяется как количество вопросов Stack Overflow с тегом этого языка за конкретный год.

Для каждого года в диапазоне 2010-2020 выполняется:

1. фильтрация вопросов по году;
2. сопоставление тегов с языками;
3. группировка по языку;
4. подсчет количества вопросов;
5. сортировка по убыванию популярности;
6. выбор top-10.

### Сохранение результата

Итоговый отчет сохраняется в формате Apache Parquet.

По умолчанию выходная папка:

```text
target/language-report-parquet
```

На Windows Spark/Hadoop часто требует `winutils.exe` для локальной записи файлов. Чтобы не требовать отдельной установки Hadoop, в проект добавлен `NoOpPermissionLocalFileSystem`, который позволяет выполнять локальную запись Parquet в рамках лабораторной работы.

## Использованные технологии

- Java 17;
- Apache Spark SQL;
- Apache Parquet;
- Maven;
- JUnit 5.

Для запуска используется локальная JDK:

```text
../../jdk-17.0.18+8
```

## Проверка работы

Добавлены следующие тесты:

- `StackOverflowXmlParserTest` - проверка разбора XML;
- `LanguageDictionaryTest` - проверка справочника языков;
- `PopularityReportServiceTest` - проверка логики построения top-10;
- `Lab2ParquetIntegrationTest` - интеграционная проверка записи и чтения Parquet.

Команда запуска:

```powershell
$env:JAVA_HOME=(Resolve-Path '..\..\jdk-17.0.18+8')
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
..\..\apache-maven-3.8.9\bin\mvn.cmd "-Dmaven.repo.local=.m2" test
```

Фактический результат последнего запуска:

```text
Tests run: 9, Failures: 0, Errors: 0, Skipped: 0
BUILD SUCCESS
```

В процессе запуска Spark может вывести предупреждение об отсутствии `winutils.exe`. Это предупреждение не мешает работе: интеграционный тест Parquet проходит успешно.

## Команда запуска приложения

Сборка:

```powershell
$env:JAVA_HOME=(Resolve-Path '..\..\jdk-17.0.18+8')
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
..\..\apache-maven-3.8.9\bin\mvn.cmd "-Dmaven.repo.local=.m2" clean package
```

## Результат работы

В результате формируется Parquet-отчет, содержащий top-10 языков программирования по каждому году с 2010 по 2020.

Отчет строится по выборке `posts_sample.xml`. Если требуется отчет по полному архиву Stack Exchange, необходимо отдельно скачать полный XML-архив и передать его приложению.

## Ограничения и замечания

В лабораторной работе используется выборка Stack Overflow, поэтому результат отражает популярность в рамках доступного файла `posts_sample.xml`.

Критерием популярности выбрано количество вопросов с тегом языка. Это простая и воспроизводимая метрика, подходящая для Spark-агрегации.

## Вывод

Лабораторная работа выполнена полностью. Реализовано Java-приложение на Apache Spark SQL, которое формирует годовые top-10 отчеты и сохраняет результат в Apache Parquet. Корректность подтверждена unit- и integration-тестами.
