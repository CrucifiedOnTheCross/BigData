# Отчет по лабораторной работе 4

## Тема

Apache ZooKeeper. Координация распределенных процессов на Java.

## Цель работы

Цель лабораторной работы - изучить ZooKeeper как сервис координации распределенных приложений и реализовать несколько примитивов синхронизации на Java.

## Постановка задачи

В лабораторной работе требуется:

- изучить запуск ZooKeeper и базовые команды CLI;
- реализовать приложение с барьерной синхронизацией;
- решить задачу обедающих философов;
- реализовать двухфазный commit-протокол для high-available регистра;
- проверить корректность работы приложения.

Программная часть выполнена на Java.

## Выбранный подход

Для решения создан Maven-проект:

```text
L4 - ZooKeeper/lab4-java
```

В проекте используется официальный ZooKeeper Java API.

Для автоматической проверки используется embedded ZooKeeper server, который поднимается прямо из тестов на свободном локальном порту. Это позволяет проверять распределенную логику без ручного запуска внешнего ZooKeeper-сервера.

Для ручного запуска приложений можно использовать отдельный ZooKeeper на `localhost:2181`.

Spring Boot не использовался, так как лабораторная работа требует консольных процессов и прямой работы с ZooKeeper API, а не веб-приложения.

## Структура проекта

Основные компоненты:

- `zoo.common` - общие утилиты подключения и работы с путями;
- `zoo.barrier` - барьерная синхронизация животных;
- `zoo.philosophers` - решение задачи обедающих философов;
- `zoo.twopc` - двухфазный commit-протокол;
- `src/test/java/zoo` - интеграционные тесты;
- `src/test/resources/logback-test.xml` - настройка логирования в тестах.

## Барьерная синхронизация

### Идея

Барьерная синхронизация нужна для того, чтобы несколько процессов начали выполнение только после того, как все участники группы готовы.

В лабораторной работе зоопарк представлен корневым znode:

```text
/zoo
```

Каждое животное создает внутри него ephemeral znode со своим именем:

```text
/zoo/lion
/zoo/tiger
/zoo/bear
```

Ephemeral znode автоматически удаляется при завершении сессии ZooKeeper.

### Реализация

Класс `Animal`:

- подключается к ZooKeeper;
- создает корневой узел `/zoo`, если он отсутствует;
- создает ephemeral znode животного;
- подписывается на изменения дочерних узлов `/zoo`;
- ожидает, пока количество участников достигнет `partySize`;
- при выходе удаляет свой znode.

Класс `AnimalRunner` позволяет запускать животное как отдельный процесс.

Пример ручного запуска трех участников:

```powershell
mvn exec:java -Dexec.mainClass=zoo.barrier.AnimalRunner -Dexec.args="lion localhost:2181 3"
mvn exec:java -Dexec.mainClass=zoo.barrier.AnimalRunner -Dexec.args="tiger localhost:2181 3"
mvn exec:java -Dexec.mainClass=zoo.barrier.AnimalRunner -Dexec.args="bear localhost:2181 3"
```

## Задача обедающих философов

### Идея

В задаче обедающих философов несколько процессов конкурируют за ограниченный набор ресурсов. Каждый философ должен получить две вилки, чтобы начать есть.

Основной риск - дедлок, когда каждый философ удерживает одну вилку и ждет вторую.

### Реализация

Для каждой вилки используется распределенная блокировка на ZooKeeper.

Блокировка реализована через ephemeral sequential znode:

```text
/philosophers/forks/fork-0/lock-0000000001
```

Процесс получает блокировку, если его sequential znode оказался первым в очереди. Если нет, он ставит watch на предыдущий znode и ждет его удаления.

Чтобы избежать дедлока, философы всегда захватывают вилки в фиксированном порядке: сначала вилку с меньшим номером, затем с большим.

Основные классы:

- `DistributedLock` - распределенная блокировка;
- `Philosopher` - логика философа;
- `PhilosopherMain` - запуск философа как отдельного процесса.

Пример ручного запуска:

```powershell
mvn exec:java -Dexec.mainClass=zoo.philosophers.PhilosopherMain -Dexec.args="plato localhost:2181 0 5 1000"
```

## Двухфазный commit-протокол

### Идея

Двухфазный commit нужен для согласованного принятия решения между несколькими участниками. Все участники должны либо применить изменение, либо откатить его.

Протокол состоит из двух фаз:

1. Голосование участников.
2. Публикация финального решения координатором.

### Структура znode

Для транзакции создается структура:

```text
/ha-register/tx/<tx-id>
/ha-register/tx/<tx-id>/votes
/ha-register/tx/<tx-id>/acks
/ha-register/tx/<tx-id>/decision
```

В корневом узле транзакции хранится предлагаемое значение регистра.

Участники публикуют голоса:

```text
COMMIT
ABORT
```

Координатор принимает решение:

- `COMMIT`, если все участники проголосовали за commit;
- `ABORT`, если хотя бы один участник проголосовал abort или истек timeout.

### Реализация

Класс `TwoPhaseCommitCoordinator`:

- создает транзакционный узел;
- ожидает голоса участников;
- публикует финальное решение.

Класс `RegisterParticipant`:

- ожидает появления транзакции;
- публикует свой голос;
- ожидает решение координатора;
- применяет новое значение только при `COMMIT`;
- публикует ack о выполнении.

Классы `CoordinatorMain` и `RegisterParticipantMain` позволяют запускать координатор и участников как отдельные процессы.

Пример ручного запуска:

```powershell
mvn exec:java -Dexec.mainClass=zoo.twopc.CoordinatorMain -Dexec.args="localhost:2181 /ha-register tx-1 3 new-value 10000"
mvn exec:java -Dexec.mainClass=zoo.twopc.RegisterParticipantMain -Dexec.args="node-a localhost:2181 /ha-register tx-1 COMMIT 10000"
```

## Проверка работы

Для проверки написаны интеграционные тесты:

- `AnimalBarrierTest` - проверяет, что животные проходят барьер только после сбора всей группы;
- `DiningPhilosophersTest` - проверяет, что философы могут поесть без дедлока;
- `TwoPhaseCommitTest` - проверяет оба исхода 2PC: `COMMIT` и `ABORT`.

Тесты самостоятельно поднимают ZooKeeper-сервер внутри JVM, выполняют сценарий и останавливают сервер.

Команда запуска:

```powershell
$env:JAVA_HOME=(Resolve-Path '..\..\jdk-17.0.18+8')
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
..\..\apache-maven-3.8.9\bin\mvn.cmd "-Dmaven.repo.local=.m2" test
```

Фактический результат последнего запуска:

```text
Tests run: 4, Failures: 0, Errors: 0, Skipped: 0
BUILD SUCCESS
```

Для ручной демонстрации можно запустить внешний ZooKeeper на `localhost:2181` и затем запускать консольные классы из проекта.
